# Consensus AI 클라우드 배치 가이드

다른 환경(다른 PC, 노트북 등)에서도 계속 개발할 수 있도록 GitHub에 올리는 방법입니다.

---

## 1. GitHub 저장소 생성

1. [github.com](https://github.com) 로그인
2. 우측 상단 **+** → **New repository** 클릭
3. Repository name: `consensus_ai` (또는 원하는 이름)
4. **Public** 선택
5. **Create repository** 클릭 (README, .gitignore는 추가하지 않음)

---

## 2. 로컬에서 Git 초기화 및 푸시

터미널에서 아래 명령어를 순서대로 실행하세요.

### 2-1. 프로젝트 폴더로 이동

```bash
cd ~/Desktop/consensus_ai
```

### 2-2. Git 초기화 (처음 한 번만)

```bash
git init
```

### 2-3. 파일 스테이징

```bash
git add .
```

### 2-4. 첫 커밋

```bash
git commit -m "Consensus AI 초기 프로젝트"
```

### 2-5. 원격 저장소 연결

> ⚠️ `YOUR_USERNAME`을 본인 GitHub 아이디로, `YOUR_REPO`를 생성한 저장소 이름으로 바꾸세요.

```bash
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
```

예시: `git remote add origin https://github.com/letsgoqc/consensus_ai.git`

### 2-6. GitHub에 푸시

```bash
git branch -M main
git push -u origin main
```

GitHub 로그인 창이 뜨면 본인 계정으로 로그인하세요.

---

## 3. 다른 환경에서 개발할 때

### 새 PC/노트북에서 처음 받기

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd YOUR_REPO
pip install -r requirements.txt
streamlit run app.py
```

### 기존 폴더에서 최신 코드 받기

```bash
cd ~/Desktop/consensus_ai
git pull origin main
```

### 변경 사항 저장 후 올리기

```bash
git add .
git commit -m "변경 내용 요약"
git push origin main
```

---

## 4. 주의사항

- **API 키는 절대 Git에 올리지 마세요.** `.gitignore`에 `.env`, `secrets/` 등이 포함되어 있습니다.
- 비밀 정보가 포함된 파일이 있다면 `git add` 전에 제거하세요.

---

## 5. Streamlit Cloud로 앱 배포 (선택)

코드만 올리는 것이 아니라, **웹에서 바로 앱을 실행**하게 하려면 [share.streamlit.io](https://share.streamlit.io)에서 Streamlit Cloud를 사용할 수 있습니다.

1. GitHub에 코드 푸시 완료
2. [share.streamlit.io](https://share.streamlit.io) 접속 → GitHub 로그인
3. **New app** → 저장소·브랜치·파일(`app.py`) 선택
4. **Advanced settings**에서 환경 변수로 API 키 입력 (선택 사항)
5. **Deploy** 클릭

배포된 앱은 `https://YOUR_APP.streamlit.app` 형식의 URL로 접속됩니다.
